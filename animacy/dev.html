Nov-2-2025

So that was a lie. I haven't been working on this consistently. In fact, I haven't been working on it at all. I was extremely burnt out, and also college continues to take up a large part of my time.
So I'll be perfectly honest. I don't know if this is ever going to be finished. I'm kind of restarting it as a project? But a few things have changed. 1. I'm rewriting it in rust. I know, rewriting things in rust is kind of a meme, but there are a few problems I want to solve and I don't know if javascript has the ability to solve them.
When I originally built this program, there were a few architectural problems with it. There was no Symbol support from the start, so it started to be an extremely daunting task to implement them. Importing and exporting never worked right. Because javascript doesn't let you properly extend built-in HTML types, I was trying to do weird workarounds with prototyping, and that was just a nightmare. Dealing with these problems requires a significant amount of rewriting.
That doesn't mean rust is a magic bullet that fixes all these problems. People rave about rust, and how cool it is, but actually working in rust is kind of a nightmare. Doing anything with graphs is really difficult because you run into wacky issues with borrowing. And a vector art program contains quite a lot of graphs. Rust is a hassle to work with. However, it is probably the least hassle of any low-level language.
C is too hands-on in terms of memory, C++ has pitifully bad error checking, Java is.. java. So that leaves Rust as the main contender.
2. I have another person working with me! Her name is Mint, and she's really enthusiastic about the project. So that's really cool, I think. And she's helping me stay on track and plan things out better than I would just talking to myself.
I can't guarantee anything. I've always been very hesitant to do so, after seeing firsthand so many horror stories of developers promising too much. But I am working on this project again. So that's something.
